Project Design

This page is dedicated to laying out the secondary design of the project. This is not intended to be a final design but an initial idea on how the game will be laid out and played.

Constants: (in JavaScript these are just unchanging variables)

BEGINNER_SIZE: The size of the grid for a beginner game. This will be set from the selected player level
INTERMEDIATE_SIZE: The size of the grid for an intermediate game. This will be set from the selected player level
ADVANCED_SIZE: The size of an advanced game. This will be set from the selected player level
BEGINNER_BOMB_COUNT: The maximum number of bombs for a beginner game.
INTERMEDIATE_BOMB_COUNT: The maximum number of bombs for a intermediate game.
ADVANCED_BOMB_COUNT: The maximum number of bombs for a advanced game.

Objects:

2Dimentioal Array
A square will have a _bomb, _adjacentBombs and a _flaged property.
  • _bomb: boolean property to determine if it holds a or not.
  • _adjacentBombs: if there is not a bomb it will hold the number of adjacent bombs
  • _flagged: tells if the sqare has been marked by a flag

2D Array object example:
_gameArray[i][j] = "true" (is a bomb);
_gameArray[i][j] = "adjacent" (is the adjacent bomb count);
_gameArray[i][j] = "flag" (this will hold a flag)

Bomb

  • _bombCount : This will count the number of bombs flagged.

Game object example:

Clock
A Game will have a _timeCount property.
  • _timeCount : This will count the time that a player uses to solve the puzzle.

Difficulty The Difficulty object will have _beginner, _intermdiate_, and _advanced properties.
  • _beginner: a integer value that will be used when calling the difficulty function when the game is being set up.
  • _intermdiate_: a integer value that will be used when calling the difficulty function when the game is being set up.
  • _advanced: a integer value that will be used when calling the difficulty function when the game is being set up.

Functions / Model:

The model for this game will use the functions listed below: Upon the page loading a beginner game will be set up. If the user chooses a different game level, then the page will update to that level, using the setLevel Function. The gameController function will be called, which sets the bombs and adjacent square numbers. The time function will start, and the game is ready for the user to play. As the user sets flags, the bomb count will go down and the square will be replaced with a flag, and will only be able to have a flag removed this will all be done with the setFlags function. When the flags match the bombs as called by the winner function, the game will end. setLevel(_BEGINNER_SIZE): This will be used to deterimine the size of the grid.

gameController(_gridSize): This will be used to set the bombs, and determine the number to be placed in the adjacent squares.

setFlags(_gameArray): This will be called when a player right clicks and places a flag or removes a flag.

clockCount(): this function will be a counter that tells how much time it has taken to complete the game.

bombCount(_flagCount): this function will show how many bombs have not been flagged.

createGrid(_gridSize, _gameArray): this function will produce the HTML for the game.

winner(): the winner function will be called when the flag count is equal to the bomb count. If the flags match the bombs the clock and game will stop.
